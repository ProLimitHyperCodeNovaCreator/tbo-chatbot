⚠️  IMPORTANT NOTES
═══════════════════════════════════════════════════════════════════════════════

1. API CREDENTIALS
   - TBO credentials (Hackathon/Hackathon@1234) are returning 401 errors
   - This is expected - credentials may be expired or invalid
   - Script automatically falls back to mock data
   
2. TO USE REAL DATA:
   - Update credentials in tbo_hotel_client.py:
     self.username = username or os.getenv('TBO_HOTEL_USERNAME', 'YOUR_USERNAME')
     self.password = password or os.getenv('TBO_HOTEL_PASSWORD', 'YOUR_PASSWORD')
   
   - Update credentials in tektravels_flight_client.py:
     self.user_id = user_id or os.getenv('TEKTRAVELS_FLIGHT_USER_ID', 'YOUR_USER_ID')
     self.password = password or os.getenv('TEKTRAVELS_FLIGHT_PASSWORD', 'YOUR_PASSWORD')

3. OPTIONAL SERVICES
   - Redis: Not running - script skips it (optional)
   - Kafka: Disabled on Windows - script skips it (optional)
   - Qdrant: Will fail if not running, but non-fatal (optional)

4. MOCK DATA VS LIVE DATA
   - "fetch_mode": "mock" = Using fallback data (APIs failed)
   - "fetch_mode": "live" = Using real API data
   - Both modes produce valid JSON with same structure
